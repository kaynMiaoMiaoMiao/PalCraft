# PalCraft 世界观设定草案

版本：0.2  
用途：为后续伙伴、生物群系事件、道具、据点设施、任务线和文案命名提供统一参考。  
原则：保持 Minecraft 生存体验的自由度，不把玩家强行塞进固定主线；世界观用于解释玩法，而不是限制玩法。

## 1. 核心概念

PalCraft 的世界建立在一种名为“灵质”的自然能量之上。灵质并不是外来魔法，而是主世界中长期存在、但很少被人系统理解的生态力量。它会在矿脉、森林、雷雨、地下洞穴、古代遗迹和生物活动密集的地方沉积，并逐渐影响周围生命。

受到灵质长期影响的生物，会产生更稳定的智慧、元素倾向、工作本能和成长潜力。这类生物被称为“伙伴”。伙伴不是单纯的怪物，也不是普通宠物；它们可以与玩家建立契约，参与战斗、探索、采集和据点生产。

玩家不是征服者，而是“契约拓荒者”：通过捕捉球建立临时灵质回路，再通过长期协作让伙伴成为队伍和据点的一部分。

## 2. 世界基调

整体基调是“温和生存 + 野外奇遇 + 据点建设”。

- 世界仍然以 Minecraft 原版生存为基础。
- 伙伴系统补充探索、战斗和自动化，不替代原版采矿、建筑和红石。
- 科技感来自灵质工程，而不是现代枪械或工业军备。
- 设定可以带有古代遗迹、生态异常和未知危险，但不走末世废土风格。
- 伙伴被设计为可合作的生态生命，避免把玩法表达成纯粹压榨劳工。

## 3. 灵质与元素

灵质会根据环境形成不同表现。PalCraft 的伙伴元素固定为九类：火、水、雷、风、木、冰、土、龙、无。

元素用于区分伙伴生态、技能风格、战斗克制和部分材料来源。初期内容可以只实装其中几类，但设计文档和数据结构应按完整九元素预留。

元素定位：

- 火：来自岩浆、烈日、燃烧和高温矿脉，代表爆发、燃烧和攻击压制。
- 水：来自河流、沼泽、海洋和雨水，代表恢复、冷却、缓速和资源运输。
- 雷：来自雷雨、高山矿脉和红石活动区域，代表速度、能量和远程打击。
- 风：来自高地、峡谷、开阔草原和气流异常区域，代表机动、闪避和范围扰动。
- 木：来自森林、作物、苔藓和温和生态区，代表恢复、采集、种植和持续作战。
- 冰：来自雪原、冻洋、高山和低温洞穴，代表控制、减速和抗压。
- 土：来自洞穴、矿物、山地和沉积层，代表防御、采矿、稳定生产。
- 龙：来自古代遗迹、深层灵质沉积和稀有生态异常，代表高阶成长和强技能。
- 无：没有明显元素倾向，适合作为通用伙伴、早期教学伙伴或特殊功能伙伴。

## 4. 元素克制关系

元素克制采用攻击方视角。基础伤害倍率为 1.0；攻击方克制防御方时倍率为 1.5；攻击方被防御方抵抗时倍率为 0.5；未列入关系的组合保持 1.0。

| 攻击元素 | 克制目标 | 被抵抗目标 |
| --- | --- | --- |
| 火 | 木、冰 | 水、土、龙 |
| 水 | 火、土 | 木、雷、龙 |
| 雷 | 水、风 | 土、木、龙 |
| 风 | 木、土 | 雷、冰、龙 |
| 木 | 水、土 | 火、冰、龙 |
| 冰 | 风、龙 | 火、水、土 |
| 土 | 火、雷 | 水、木、风 |
| 龙 | 火、水、雷、风、木 | 冰、土、龙 |
| 无 | 无 | 无 |

无属性不参与克制，也不抵抗任何元素。它的价值来自稳定、通用和低理解成本，而不是克制优势。

龙属性定位为稀有高阶元素。龙属性技能对常规自然元素有压制力，但会被冰、土和龙属性抵抗；冰属性是当前体系中最明确的反龙手段。

## 5. 伙伴的来源

伙伴最早是自然生物与灵质长期共生后的结果。它们通常具有以下特征：

- 保留动物或自然现象的直觉。
- 拥有可识别的元素倾向。
- 拥有独立的天赋值，决定同种伙伴之间的成长差异。
- 对据点核心、捕捉球和灵质工作站有反应。
- 能理解简单指令，但仍保留自己的生态习性。

伙伴不是凭空刷新出来的商品。后续设计自然生成时，可以让不同伙伴与环境绑定：

- 火属性伙伴更常出现在沙漠边缘、岩浆湖、下界入口附近或燃烧区域附近。
- 水属性伙伴更常出现在河岸、沼泽、海边浅滩和雨后区域。
- 雷属性伙伴更常出现在雷雨后的高地、红石矿附近或开阔草原。
- 风属性伙伴更常出现在山脊、峡谷、草原和高空平台附近。
- 木属性伙伴更常出现在森林边缘、花田、作物田附近。
- 冰属性伙伴更常出现在雪原、冻河、冰刺平原和高山冻土。
- 土属性伙伴更常出现在山地、洞穴入口、裸露矿脉附近。
- 龙属性伙伴更常与古代遗迹、特殊结构、稀有天气或高难事件绑定。
- 无属性伙伴可以分布在多种常规生态中，承担早期教学和泛用功能。

## 6. 天赋系统

每个伙伴实例都拥有一个天赋值。天赋值是伙伴出生或生成时固定的个体潜力，范围为 0.00 到 100.00，精确到小数点后两位。

天赋规则：

- 天赋越高，伙伴的基础属性越高。
- 天赋越高，伙伴升级时获得的成长属性也越高。
- 天赋属于同种伙伴之间的个体差异，不改变伙伴物种、元素或技能定位。
- 天赋暂时不能通过道具、训练、工作站或指令修改。
- 捕捉前后天赋保持一致，不能因为召唤、收回、存档或分配据点而变化。

推荐实现方向：

- 伙伴生成时随机生成 `talent`，存储为 0.00 到 100.00 的数值。
- 捕捉成功后将 `talent` 写入 `PalInstance`。
- 属性计算时将天赋作为乘区或加成因子，而不是直接替换物种基础属性。
- UI 中可以显示天赋，但前期不提供洗练、重置或强化入口。

世界观解释：

天赋代表伙伴体内灵质回路的稳定度和上限。高天赋伙伴不是另一个物种，而是同种伙伴中灵质回路更完整、成长更顺畅的个体。

## 7. 捕捉球设定

捕捉球是玩家掌握的第一种灵质工具。它不是把伙伴永久封印进容器，而是在命中后尝试建立一段“契约回路”。

捕捉成功时：

- 捕捉球记录伙伴的灵质印记、元素、天赋和基础状态。
- 伙伴实体暂时离开世界，进入玩家的伙伴数据仓库。
- 玩家之后可以通过召唤，将伙伴重新投射到现实位置。

捕捉失败时：

- 契约回路无法稳定。
- 伙伴会被短暂扰动，但不会复制、分裂或丢失数据。

这个设定可以解释为什么低生命值更容易捕捉：不是因为伙伴被强行打服，而是因为激烈战斗后它的灵质防御不稳定，更容易被捕捉球建立回路。

## 8. 据点核心设定

据点核心是一种稳定灵质的锚点。玩家放置据点核心后，周围区域会形成一个低强度的“协作场”。这个场不会改变地形，也不会阻止原版玩法，只用于让伙伴识别工作范围、存储目标和任务优先级。

据点核心承担三个功能：

- 记录据点归属和范围。
- 让伙伴知道哪些方块、箱子和工作站属于当前据点。
- 将玩家的指令转换成伙伴可以执行的任务队列。

后续据点工作站只承担资源加工、采集辅助或产出引导，不承担通用合成职责。所有常规配方都通过原版工作台直接合成，不设置合成时间、装配队列或通用装配台。

后续据点工作站可以解释为不同类型的灵质转换器：

- 矿物加工台：把矿物灵质稳定为矿石、碎片或加工材料。
- 木材加工台：引导木属性伙伴处理原木、木板和树苗资源。
- 石材加工台：处理石材、圆石、矿物副产物。
- 食物料理台：处理作物、肉类和恢复类物品。

## 9. 玩家身份

玩家可以被设定为偶然发现灵质技术的拓荒者。这个身份足够宽，不会与玩家自定义角色冲突。

推荐称呼：

- 契约拓荒者
- 灵质工匠
- 伙伴训练者
- 据点管理者

不建议把玩家设定成唯一救世主。更适合让玩家在世界中逐渐发现灵质、建立伙伴关系、扩展据点，并处理越来越复杂的生态异常。

## 10. 世界冲突

PalCraft 不需要一开始就做大型剧情，但需要有可扩展的冲突来源。

推荐冲突：

- 野外灵质过载：某些区域的伙伴变得不稳定，攻击性增强。
- 古代设施失控：遗迹中的灵质设备仍在运行，吸引或扭曲附近生物。
- 据点资源压力：玩家需要在探索、采集、生产和防御之间分配伙伴。
- 稀有伙伴保护：部分伙伴只在特殊天气、时间或地形出现。
- 生态平衡：过度采集某些资源会让特定伙伴刷新变少，后续可作为高级系统考虑。

这些冲突可以逐步转化为 Boss、事件、稀有掉落、任务线或世界生成内容。

## 11. 基础伙伴组

第一批基础伙伴覆盖火、水、雷、风、木、冰、土七个自然元素。它们主要用于验证捕捉、召唤、成长、技能、元素克制和据点工作适性。龙属性暂时保留给后续稀有伙伴或事件伙伴；无属性可以后续作为教学伙伴或通用伙伴补充。

| 伙伴 | 元素 | 定位 | 生态位置 | 工作适性 |
| --- | --- | --- | --- | --- |
| 小火苗 | 火 | 近中距离爆发，适合教学火焰伤害 | 沙漠边缘、岩浆湖附近、燃烧遗迹附近 | 料理、熔炼辅助、战斗防卫 |
| 水精灵 | 水 | 恢复、缓速和稳定续航 | 河岸、沼泽、海边浅滩、雨后区域 | 料理、种植辅助、运输 |
| 电星仔 | 雷 | 速度、远程能量弹和早期战斗入门 | 开阔草地、高地、红石矿附近、雷雨后区域 | 搬运、轻量采矿辅助、能量类设施互动 |
| 风鸟龙 | 风 | 高机动、扰动和追击 | 山脊、峡谷、草原、高空平台附近 | 探索辅助、搬运、巡逻防卫 |
| 小树酱 | 木 | 恢复、持续作战和种植入门 | 森林边缘、花田、作物田附近 | 种植、采集、恢复辅助 |
| 冰莱姆 | 冰 | 减速、控制和抗压 | 雪原、冻河、冰刺平原、高山冻土 | 冷却、储存辅助、防卫 |
| 泥佬巴 | 土 | 高防御、采矿和据点承压 | 山地、洞穴入口、裸露矿脉附近 | 采矿、搬运、据点防卫 |

### 小火苗

小火苗是由高温灵质凝结出的轻型火属性伙伴。它们通常围绕热源活动，情绪激动时头顶的火苗会变亮。

- 元素：火
- 性格：冲动、亲近热源、容易被战斗吸引
- 战斗：使用火花撞击和短距离火焰喷吐
- 掉落倾向：火绒灰、微燃灵质碎片

### 水精灵

水精灵由湿润环境中的灵质水滴聚合而成，行动安静，通常会跟随水流和雨声移动。

- 元素：水
- 性格：温和、警觉、喜欢靠近水源
- 战斗：使用水弹、缓速水流和低强度恢复技能
- 掉落倾向：清露珠、水纹灵质碎片

### 电星仔

电星仔是早期最适合作为教学伙伴的生物。它体内有一枚类似电荷结晶的灵质核心，会把多余能量释放成细小电弧，因此在夜晚或雨后更容易被发现。

- 元素：雷
- 性格：好奇、警觉、容易被红石信号和雷雨吸引
- 战斗：使用撞击和短距离雷能弹
- 掉落倾向：雷光晶粒、带电灵质碎片

后续可扩展变体：

- 高地电星仔：速度更快，掉落更多雷属性材料。
- 雷鸣电星仔：稀有天气变体，技能更偏远程。
- 工坊电星仔：据点长期工作后可能获得的外观或特性变体。

### 风鸟龙

风鸟龙是拥有细长翼膜和轻型骨架的风属性伙伴。它的名字中带“龙”，但基础版本不归入稀有龙属性；它更接近受风灵质影响的鸟形伙伴。

- 元素：风
- 性格：敏捷、好动、喜欢停在高处观察
- 战斗：使用俯冲撞击、风刃和短暂击退
- 掉落倾向：轻羽、流风灵质碎片

### 小树酱

小树酱是森林和农田附近常见的木属性伙伴。它们会把灵质储存在嫩叶和根须中，适合承担早期种植与恢复定位。

- 元素：木
- 性格：温和、慢热、喜欢靠近作物和花丛
- 战斗：使用藤蔓拍击、种子弹和轻微恢复
- 掉落倾向：苔纹纤维、嫩枝灵质碎片

### 冰莱姆

冰莱姆是低温环境中的半流体冰属性伙伴，身体像带霜晶的凝胶，会在移动时留下短暂的冷雾。

- 元素：冰
- 性格：迟钝、耐心、对热源敏感
- 战斗：使用冰粒喷射、减速撞击和短时间控制
- 掉落倾向：霜凝胶、冰纹灵质碎片

### 泥佬巴

泥佬巴是土属性伙伴，身体由湿土、砂石和稳定灵质混合形成。它行动不快，但很耐打，适合作为早期据点的采矿和防卫伙伴。

- 元素：土
- 性格：稳重、迟缓、喜欢洞穴和矿脉
- 战斗：使用泥块投掷、重击和短暂防御强化
- 掉落倾向：稳定石核、泥质灵质碎片

## 12. 命名风格

中文命名建议短、清楚、有 Minecraft 物品感。

伙伴命名：

- 使用 2 到 4 个汉字，便于 UI 显示。
- 名字最好同时暗示外形和元素，例如小火苗、水精灵、电星仔、风鸟龙、小树酱、冰莱姆、泥佬巴。
- 避免直接复用其他商业作品中高识别度的词、角色名或技能名。

道具命名：

- 基础道具直白命名：捕捉球、据点核心、灵质碎片。
- 进阶道具体现功能：强化捕捉球、稳定芯片、元素导管。
- 不要过早引入太多专有名词，避免玩家理解成本过高。

设施命名：

- 以用途命名优先：矿物加工台、木材加工台、食物料理台。
- 后续高级设施再加入世界观词汇：灵质调谐器、契约终端、生态扫描仪。
- 不使用通用装配台作为合成入口，避免与原版工作台职责重叠。

## 13. 第一批内容建议

伙伴：

- 小火苗：火属性，早期爆发输出，适合引入燃烧和火属性克制。
- 水精灵：水属性，恢复和缓速定位，适合料理、种植辅助和运输。
- 电星仔：雷属性，新手伙伴，战斗和能量互动入门。
- 风鸟龙：风属性，高机动追击伙伴，适合探索、搬运和巡逻防卫。
- 小树酱：木属性，小型温和伙伴，擅长种植、采集和恢复。
- 冰莱姆：冰属性，控制和减速定位，适合防卫和冷却类设定。
- 泥佬巴：土属性，低速高防，擅长采矿、搬运和据点防卫。

材料：

- 灵质碎片：基础通用材料，来自伙伴掉落、工作站或遗迹箱子。
- 微燃灵质碎片：火属性材料，用于火属性技能和料理相关设施。
- 水纹灵质碎片：水属性材料，用于恢复、料理和运输相关设施。
- 雷光晶粒：雷属性材料，用于雷属性技能和能量设施。
- 流风灵质碎片：风属性材料，用于速度、探索和搬运相关设施。
- 苔纹纤维：木属性材料，用于恢复、种植和料理。
- 冰纹灵质碎片：冰属性材料，用于控制、防卫和冷却相关设施。
- 稳定石核：土属性材料，用于据点核心和工作站升级。

据点设施：

- 据点核心：建立据点范围。
- 矿物加工台：生成或加工矿物类任务。
- 木材加工台：生成或加工木材类任务。
- 食物料理台：生成基础食物和恢复类物品。

合成规则：

- 捕捉球、据点核心、工作站、升级部件和常规材料配方都通过原版工作台即时合成。
- 不新增通用装配台。
- 不设计合成耗时、合成队列或需要伙伴驻守的装备合成流程。
- 据点自动化负责资源获取、加工辅助和存储流转，不替代玩家的原版合成入口。

## 14. 后续开发落点

短期开发可以先让世界观服务于实际系统：

1. 为新增伙伴建立统一模板：生态位置、元素、技能、工作适性、掉落物、天赋范围。
2. 为九元素建立统一枚举、语言文件和技能倍率计算。
3. 为伙伴实例补充 `talent` 字段，并在属性计算中体现基础属性和成长属性加成。
4. 为据点工作站建立统一命名和功能边界，但不加入通用装配台。
5. 为捕捉球等级和材料来源设计原版工作台配方链。
6. 在语言文件中统一使用“伙伴”，避免“宠物”“随身宠物”混用过多。
7. 后续需要任务线时，从“灵质异常”和“古代设施”两个方向展开。

这份设定不是最终剧情文本，而是内容设计约束。后续每新增一个伙伴、道具或设施，都应能回答三个问题：它来自什么环境、它为什么会有这个能力、它在玩家玩法中解决什么问题。
