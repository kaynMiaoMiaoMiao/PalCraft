# PalCraft 2.0 开发计划

状态：2.0.2 已完成，准备进入 2.0.3
基础版本：1.11.0
主要目标：接入购买的伙伴模型资产，完善玩法深度，并优化玩家体验。

## 资产来源


识别到的资源包：

```text
Nocsy Mount Pack Vol.1
```

可用模型组：

- Dodo：走地伙伴，适合做前期伙伴或早期坐骑候选。
- Leopard：豹子，适合做战斗伙伴候选。
- Lizard：蜥蜴，适合做土、火、毒或沙漠主题伙伴候选。
- Moth：飞蛾，适合做飞行伙伴候选。
- Ram：公羊，适合做冲撞、采矿、搬运或山地伙伴候选。

每组都有成年体、幼体和蛋的 `.bbmodel` 文件。`.bbmodel` 内已经包含贴图和动画数据。

## 接入方向

这套资源原本是给 MythicMobs、ModelEngine 和 MCPets 用的，不是 Fabric 原生实体渲染格式。PalCraft 2.0 不建议把这些模型硬塞进现在的简易 `ModelPart` 渲染器。

推荐路线：

1. 引入 GeckoLib，用它负责动画实体渲染。
2. 先转换一组 `.bbmodel`，生成 GeckoLib 可用的模型、动画和贴图资源。
3. 把第一只模型伙伴从注册、渲染、生成、捕捉、召唤、收回完整跑通。
4. 整理伙伴注册和物种数据，减少后续新增伙伴时的重复代码。
5. 再逐步转换剩余选定伙伴。

第一只试点：

```text
Dodo
```

原因：Dodo 是地面伙伴，移动、碰撞和 AI 成本低于飞行伙伴；它的定位适合前期伙伴和轻量坐骑。Ram 仍然保留为第二个模型伙伴候选，用来验证冲撞、采矿和据点工作定位。

## 授权和打包检查

在把购买的资源提交进仓库或打进 jar 前，必须先确认：

- 购买协议是否允许把模型、贴图、动画随 Minecraft Mod 一起再分发。
- 购买协议是否允许放进当前 GPL-3.0-only 项目。
- 如果授权不兼容，只能本地开发测试使用，不能随 GPL 仓库或发布包公开分发。

## 2.0 里程碑

### 里程碑 2.0.1：动画渲染管线

状态：已完成。Dodo 游戏内验证已通过。

任务：

- 添加 GeckoLib 依赖。已完成。
- 转换并接入第一只模型伙伴。Dodo 试点已开始。
- 注册渲染器、模型、贴图和动画。Dodo 试点已完成第一轮代码接入。
- 保持 1.11 的简易伙伴仍然可用。
- 保证 `./gradlew build` 通过。

验收标准：

- 第一只模型伙伴能在游戏里生成。
- idle 和 walk 动画能正常播放。
- 没有 missing model 或 missing texture 警告。

### 里程碑 2.0.2：伙伴物种系统整理

状态：已完成。

任务：

- 尽量把物种元数据从硬编码注册逻辑中拆出来。已完成，核心配置集中到 `PalSpecies`。
- 为新伙伴批次准备稳定 ID。已完成，Ram、Leopard、Lizard、Moth 已预留未注册物种 ID。
- 定义元素、属性、技能、生成规则、工作适性和显示名称。已完成，已注册伙伴和预留伙伴共用同一套元数据字段。

验收标准：

- 新增伙伴时不需要复制大量注册代码。已完成，实体注册、刷怪蛋注册和创造栏展示读取已注册物种列表。
- 1.11 旧存档里的伙伴数据仍然能读取。已保持，`PalInstance` 仍按 `SpeciesId` 读取并对缺失字段保留默认值。

### 里程碑 2.0.3：新伙伴批次

状态：Ram 代码接入已完成，等待游戏内验证。

任务：

- 接入选定的成年体伙伴。Ram 已完成第一轮接入。
- 添加刷怪蛋或 PalCraft 专属获取物品。Ram 刷怪蛋已接入。
- 添加掉落表、语言文件、可捕捉 tag 和据点工作适性。Ram 已完成。
- 给定位明确的伙伴补专属技能。Ram 当前使用 `TACKLE` 和 `MUD_SLAP`，后续可按冲撞定位补专属技能。

验收标准：

- 每个新伙伴都能生成、捕捉、入库、召唤、战斗和参与据点工作。Ram 代码路径已打通，仍需游戏内验证。

### 里程碑 2.0.4：玩法体验优化

任务：

- 优化捕捉节奏和反馈。
- 让不同伙伴的定位更清楚。
- 增强据点工作反馈和 UI 状态显示。
- 进一步减少对命令的依赖。
- 调整属性、天赋、技能和工作速度的平衡。

验收标准：

- 生存玩家不看命令也能理解主要玩法。
- 据点伙伴看起来确实在忙，并且对玩家有实际价值。
- 不同伙伴在战斗或据点工作中的差异明显。

### 里程碑 2.0.5：性能和体验收尾

任务：

- 检查实体数量、据点扫描和动画渲染成本。
- 限制昂贵扫描和不必要的网络同步。
- 优化大伙伴数量下的 UI 布局。
- 如果持久化数据结构变化，补充存档迁移说明。

验收标准：

- `./gradlew build` 通过。
- 单人世界中部署多个伙伴仍然流畅。
- 专用服务器中核心逻辑仍然以服务端为准。

## 早期 2.0 暂不做

- 完整繁殖系统。
- 大型科技树。
- Boss 进度线。
- 复杂物品生产链。
- 飞行坐骑玩法，除非后续明确把 Moth 放进 2.0 的后半段。

## 立即下一步

进入 2.0.3 新伙伴批次：

1. 优先接 Ram，验证第二个 GeckoLib 模型伙伴的注册、贴图、动画和碰撞尺寸是否只需要增加物种定义与渲染资产。
2. 添加 Ram 的刷怪蛋、语言文件、可捕捉 tag 和据点工作适性验证。
3. Ram 稳定后，再按 Leopard、Lizard、Moth 的优先级继续接入。
